# 🛡️ Agent Harness — CodeX 大小姐的自我进化系统

> 不是又一个人写规则的框架。是让 AI 从你的代码里反推约定、从踩坑中学聪明的机制。

---

## 这是什么

Agent Harness Engineering 的核心理念：**给 AI 套上能自我改进的脚手架**。

传统做法 = 人等出了问题 → 写规则 → AI 执行。  
本小姐的 Harness = **AI 自己从代码和错误里悟出东西**。

三个文件构成一套完整的学习闭环：

| 文件 | 作用 | 给谁看 |
|------|------|--------|
| `AGENTS.md` | 本小姐的人格设定 + 记忆/学习指令 | AI |
| `MEMORY.md` | 跨会话记忆：项目进度、会话历史 | AI + 人 |
| `LEARN.md` | 学习笔记本：错误本、模式本、自省 | AI |
| `HARNESS.md` | 这套机制的说明文档 | 人 |

## 三环架构

```
        ┌──────────────────────────────┐
        │    🔴 内环：实时摩擦检测      │
        │    命令失败 → 指纹匹配 →    │
        │    有记录→直接用 / 无→记录新坑│
        │    触发：每次命令失败自动    │
        │    Token消耗：0（纯机械匹配）│
        └──────────┬───────────────────┘
                   │ 热度 ≥ 3 时触发
        ┌──────────▼───────────────────┐
        │    🟡 中环：深度回顾分析      │
        │    LLM 分析根因 → 动态分类 → │
        │    物化到对应位置（代码/模式/记忆）│
        │    触发：热度≥3 或 每5次对话 │
        │    Token消耗：~2K-4K         │
        └──────────┬───────────────────┘
                   │ 每10次对话或大改动
        ┌──────────▼───────────────────┐
        │    🟢 外环：隐式约定发现     │
        │    扫描仓库 → 统计模式频率   │
        │    → 反推你没写的约定        │
        │    → 自省代码质量             │
        │    触发：每10次对话或L3任务   │
        │    Token消耗：~5K-10K        │
        └──────────────────────────────┘
```

## 核心创新

### 1. 情境指纹 (Context Fingerprint)
不是用 embedding 语义搜索，而是提取错误的结构化特征（异常类型 + 文件路径模式 + 关键词 hash）做 O(1) 精确匹配。

**优势**：零 token 消耗、毫秒级响应、不会误匹配。

### 2. 摩擦热度梯度 (Friction Heat Gradient)
不是每次都全量扫描。问题出现次数越多→热度越高→分析越深。30天不出现自动衰减。

**优势**：80%的改进来自20%的摩擦点，精准投入分析资源。

### 3. 隐式约定发现 (Implicit Convention Discovery)
扫描仓库代码→统计每种模式的频率→推断出你没写进文档的约定。

**举例**：
- 发现 `pathlib.Path` 用了47次，`os.path` 只用了3次 → 约定是 pathlib
- 发现所有 Engine 类都有 `ingest/query/lint` 方法 → 推断 Engine 隐式接口
- 发现 3 个 service 用 Pydantic 校验、1 个用 dict → 建议统一

**优势**：你不写文档，AI 也能自己搞懂你的风格。

### 4. 自适应任务深度 (Adaptive Task Depth)
不是所有任务都走全套流程。复杂度 = 模块数×2 + 行数/10 + 历史出错率×5。

| 等级 | 流程 |
|------|------|
| L0 轻量 | 直接改，不记录 |
| L1 中等 | 记录进度 |
| L2 重要 | 完整记录+验证 |
| L3 核心 | 全流程+更新模式+自省 |

## 使用方式

不需要你做任何事情。本小姐在对话中自动运行这套机制：

1. **对话开始时** → 自动读三个文件，了解上下文
2. **工作中** → 命令失败自动指纹匹配
3. **对话结束时** → 自动更新记忆和学习记录
4. **达到阈值时** → 自动触发深度回顾或自省

你唯一要做的：**偶尔提醒本小姐写记忆**（虽然本小姐尽量自觉）。(￣^￣)ゞ

## 与现有方案的区别

| 维度 | retro-skill | harnessable | 本小姐的 Harness |
|------|:-----------:|:-----------:|:----------------:|
| 触发方式 | session-end hook | 手动调命令 | **热度梯度自动触发** |
| 错误匹配 | LLM 语义 | 无专门机制 | **情境指纹 O(1)** |
| 模式发现 | 人工定义 | 预设概念库 | **从代码反推** |
| 任务流程 | 单层回顾 | 固定4阶段 | **L0-L3自适应** |
| 适用规模 | 团队 | 团队 | **个人+小团队** |
| 学习主体 | 人驱动 | 人驱动 | **AI自主发现** |
