# 使用指南

## 快速开始

### 1. 启动服务

在终端执行以下命令启动系统：

```bash
cd CodeXFiles/backend
python main.py
```

启动成功后终端会显示：

```
🚀 Wiki-RAG 系统启动中...
   LLM Provider: deepseek-chat
   API 文档: http://0.0.0.0:8000/docs
   🖥️  前端页面: http://0.0.0.0:8000/
```

### 2. 访问前端页面

打开浏览器访问 **http://localhost:8000/** 即可进入聊天界面。

### 3. 开始提问

在输入框中输入问题，选择查询模式，点击发送或按 Enter 键。

## 查询模式选择

系统提供三种查询模式，适用于不同场景：

### 🤖 Auto 模式（推荐）

**自动选择模式**，系统会根据问题特征自动选择最优引擎：

- **短问题（≤50字）**：使用 Pipeline 模式，优先 Wiki 精确命中
- **长问题（>50字）**：使用 Agent 模式，让 AI 自主决策
- **疑问句**：自动使用 Agent 模式

**适用场景**：日常使用，无需手动切换

### 🧠 Agent 模式

**ReAct Agent 自主决策模式**，AI Agent 会自行决定查询策略：

1. Agent 分析问题意图
2. 决定使用 Wiki 还是 RAG
3. 评估回答质量
4. 必要时补充查询

**适用场景**：
- 复杂多跳问题（如"今年新入职的律师有哪些？他们主要负责什么业务？"）
- 模糊问题（需要 Agent 推理和追问）
- 需要综合分析的问题

### 🔗 Pipeline 模式

**固定流水线模式**，严格按照 Wiki → RAG 的路径执行：

1. 先查询 Wiki 知识库
2. Wiki 命中则直接返回
3. Wiki 未命中则走 RAG 检索 + LLM 生成

**适用场景**：
- 简单事实性问题（如"公司的成立时间？"）
- 需要精确答案的问题
- 测试和调试

## 文档导入

### 支持的格式

| 格式 | 说明 | 示例 |
|------|------|------|
| PDF | 扫描件或电子文档 | 合同、案例汇编 |
| Word (.docx) | Microsoft Word 文档 | 周报、月报、报告 |
| Markdown (.md) | 轻量级标记文件 | Wiki 页面、技术文档 |
| TXT | 纯文本文件 | 笔记、清单 |

### 导入方式

通过 API 或管理界面上传文件：

```bash
curl -X POST http://localhost:8000/api/ingest \
  -F "file=@文档.pdf"
```

导入后的文档会自动：
1. 分块处理（每块 500 字符，重叠 50 字符）
2. 向量化存入 ChromaDB
3. 可供 RAG 检索查询

## Wiki 知识库

### 页面结构

每个 Wiki 页面遵循以下结构：

```markdown
---
title: 页面标题
type: entity
created: 2026-05-17
updated: 2026-05-17
tags: [tag1, tag2]
---

# 页面标题

## 概述

正文内容...

## 详细说明

详细内容...

## 相关文档

- 详见 [[项目概述]] 了解项目背景
```

### 交叉引用

使用 `[[页面名称]]` 语法创建页面间的交叉引用，例如：

```
详见 [[系统架构]] 了解设计细节
详见 [[RAG引擎]] 了解检索机制
```

### 页面类型

| 类型 | 用途 | 示例 |
|------|------|------|
| entity | 实体/对象描述 | 项目、系统、功能 |
| concept | 概念定义 | OKR、RAG、双引擎 |
| source-summary | 外部来源总结 | 周报摘要、会议记录 |
| comparison | 对比分析 | DeepSeek vs GPT |
| synthesis | 综合总结 | 年度技术总结 |

## 常见问题

### Q: 系统知识库是空的，怎么办？
文档为空时系统会通过 RAG 的 LLM 自身知识尽力回答。建议先导入一批文档，让知识库有内容可查。随着使用，高质量问答会自动沉淀到 Wiki。

### Q: 如何判断答案是否来自文档？
检查返回结果中的 `source` 字段：
- `wiki`：来自 Wiki 知识库（高置信度）
- `rag`：来自 RAG 检索 + LLM 生成
- `agent`：Agent 自主决策的结果

### Q: 回答不准确怎么办？
- 切换模式试试（Auto → Agent）
- 补充更多相关文档
- 检查问题表述是否清晰

### Q: 如何查看系统状态？
访问 `GET /api/health` 查看：
- LLM 是否正常连接
- Wiki 页面数量
- 向量数据库中的文档数
- 系统运行时间
