# IT 安全规范

## 总则

为保障公司信息安全，防范数据泄露和网络攻击，特制定本规范。本规范适用于公司全体员工、实习生、外包人员及第三方合作伙伴。

## 账户与密码管理

### 密码策略

| 要求 | 标准 |
|------|------|
| 最小长度 | 12 位 |
| 复杂度 | 必须包含大写字母、小写字母、数字和特殊符号中的至少三种 |
| 有效期 | 每 90 天强制更换 |
| 更换限制 | 新密码不得与最近 5 次历史密码相同 |
| 登录失败锁定 | 连续 5 次失败锁定 30 分钟 |

### 账户管理

- 员工入职时由 IT 部门统一创建账户，默认密码需首次登录修改
- 员工离职时 IT 部门应在 4 小时内禁用其所有账户
- 禁止共享账户，每个账户仅限本人使用
- 第三方人员的临时账户有效期最长 30 天，到期自动失效

## 数据安全

### 数据分类

| 级别 | 定义 | 示例 |
|------|------|------|
| 公开 | 可对外公开发布的信息 | 公司简介、新闻稿 |
| 内部 | 仅限内部使用，不得外传 | 内部制度、通讯录 |
| 敏感 | 泄露会造成一定损失 | 客户信息、项目方案 |
| 机密 | 泄露会造成重大损失 | 财务数据、核心技术、法律案件材料 |

### 数据保护要求

1. **敏感及以上数据**必须加密存储（AES-256）
2. **机密数据**传输必须使用 HTTPS/TLS 1.2+
3. **客户信息**（姓名、电话、身份证等）不得通过明文邮件传输
4. 禁止将公司数据上传至未授权的第三方云存储
5. 数据备份策略：
   - 核心系统：每日全量备份 + 每小时增量备份
   - 备份数据保留周期：日备份保留 30 天，月备份保留 12 个月
   - 备份数据异地存储，与主数据中心物理隔离

## 网络安全

### 办公网络

- 办公网络划分为内部办公网和访客网络，两个网络物理隔离
- 内部网络接入需通过 MAC 地址认证
- 访客网络仅提供互联网访问，不得访问内部系统

### 远程访问

- 远程办公必须通过 VPN 接入公司内网
- VPN 连接使用双因素认证（密码 + 动态令牌）
- 远程访问会话空闲超过 15 分钟自动断开
- 禁止通过公共 Wi-Fi 连接 VPN

### 防火墙规则

```
入站规则：
  - 仅开放 80（HTTP）、443（HTTPS）、22（SSH 指定 IP）等必要端口
  - 所有入站流量默认拒绝
出站规则：
  - 默认允许 HTTP/HTTPS
  - 以下协议默认拒绝：Telnet、FTP、SMB（文件共享）
  - 出站 DNS 仅允许公司 DNS 服务器
```

## 设备管理

### 办公设备

- 公司配发的电脑、手机等设备仅限工作使用
- 设备必须安装统一的安全客户端（EDR）
- 禁止私自安装未经 IT 部门审批的软件
- 禁止将公司设备连接至非安全的公共 Wi-Fi

### 移动存储

- 禁止使用个人 U 盘、移动硬盘连接公司电脑
- 公司发放的加密 U 盘仅限在内部网络使用
- 涉密文件禁止通过 U 盘、光盘等介质带出办公区域

## 软件安全

### 软件安装规范

| 软件类别 | 审批流程 | 示例 |
|---------|---------|------|
| 办公软件 | 部门主管审批 | Office、WPS、钉钉 |
| 开发工具 | IT 部门审批 | VS Code、Git、Docker |
| 第三方 SDK | 安全评审 | API SDK、开源组件 |
| 云服务 | 法务+安全双审批 | 云存储、PAAS 服务 |

### 漏洞管理

- 操作系统安全补丁：关键补丁 48 小时内部署
- 高危漏洞：72 小时内修复
- 中危漏洞：15 天内修复
- 低危漏洞：30 天内修复

## 应急响应

### 安全事件分级

| 级别 | 定义 | 响应时间 |
|------|------|---------|
| P0 | 系统瘫痪、数据大规模泄露 | 15 分钟内响应 |
| P1 | 高危漏洞被利用、部分数据泄露 | 1 小时内响应 |
| P2 | 中危安全事件 | 4 小时内响应 |
| P3 | 低危安全事件 | 24 小时内响应 |

### 报告机制

发现任何安全异常，立即通过以下渠道报告：
- 紧急热线：xxx-xxxx-xxxx
- 邮件：security@company.com
- 钉钉：IT 安全群 @安全管理员

报告后按 [事件响应流程](../IT安全规范.md#应急响应) 逐级升级处理。
